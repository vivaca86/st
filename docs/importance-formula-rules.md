# 필기 중요도 연결 및 공식·이론 집계 규칙

작성일: 2026-07-15  
조사 대상: `Backup 2026-07-15/전산기`의 GoodNotes 108개  
원본 변경: 없음

## 결론

필기의 글씨를 완벽히 해독하지 못해도, **같은 문서 안의 페이지 순서, GoodNotes 페이지 UUID, 필기에 적힌 문제번호, 표시의 좌표**를 결합하면 중요 문제를 DB 문제와 연결할 수 있다.

가장 안전한 연결 키는 다음 순서다.

1. 과목과 원본 문서
2. GoodNotes 페이지 UUID와 문서 내 페이지 순서
3. 대응하는 원본 PDF 페이지
4. 필기에서 인식된 문제번호
5. 원본 PDF 페이지에 실제 존재하는 문제번호
6. 별표·`중요`·`잘 나옴` 등의 위치와 문제번호 사이의 거리

`외우자`와 `잘 나온다`는 같은 의미가 아니다. 따라서 한 개의 중요도 숫자로 바로 합치지 말고, 최소한 다음 세 축을 따로 저장한 뒤 화면에서 종합 점수를 계산해야 한다.

- `frequency_signal`: 잘 나옴, 자주 나옴, 빈출, 출제 빈도
- `memorization_signal`: 외우자, 암기, 답만 외우기
- `negative_signal`: 잘 안 나옴, 시험에 안 나옴, 삭제, 빼자

## GoodNotes 구조에서 확인한 연결 근거

### 1. 페이지 UUID는 전부 추적 가능하다

- 필기 인식 데이터가 있는 `search/<UUID>`는 877개다.
- 877개 모두 같은 문서의 `index.notes.pb` 페이지 순서에서 UUID를 찾을 수 있었다.
- 인식 단위에는 후보 글자뿐 아니라 관찰된 구조상 글자의 사각형 위치와 관련 필기 획 UUID도 들어 있다.
- 따라서 `필기 문구가 어느 페이지의 어느 위치에 있었는지`를 DB 근거로 보존할 수 있다.

### 2. 원본 문제 PDF와 필기 페이지의 관계

필기 인식 엔트리 877개를 페이지 위치로 분류하면 다음과 같다.

| 유형 | 필기 페이지 수 | 연결 방법 |
|---|---:|---|
| 원본 PDF 페이지 위에 직접 쓴 필기 | 145 | 필기 페이지 순서가 곧 PDF 페이지 번호 |
| PDF 뒤에 같은 순서로 붙인 별도 필기 페이지 | 701 | `필기 페이지 순서 - PDF 페이지 수`를 대응 PDF 페이지 후보로 사용 |
| PDF 대응 범위를 한 쪽 넘은 추가 필기 | 1 | 필기 문제번호로 원본 페이지를 재검색 |
| 대응 PDF가 없는 순수 필기 노트 | 30 | 문제번호·과목·내용으로 별도 매칭하거나 검수 |

즉 877개 중 846개(96.47%)는 구조만으로 원본 PDF 페이지를 직접 가리키거나 같은 순서의 대응 페이지 후보를 만든다. 별도 필기 페이지 방식은 문제번호 대조로 최종 확정한다.

한 PDF 페이지에는 여러 문제가 있으므로 `PDF 한 쪽 = 문제 하나`로 처리하면 안 된다. PDF 페이지를 먼저 찾은 다음 그 안의 문제번호 단위로 다시 분리해야 한다.

### 3. 실제 문제번호 대조 표본

| 문서 | 대응 PDF 쪽 | PDF에 있는 문제번호 | 필기에서 읽힌 문제번호·표시 |
|---|---:|---|---|
| `전기기기/1.직류기1` | 5 | 21~24 | 23, 24, `그냥 외우는 문제`, `잘 안 나옴` |
| `전기기기/1.직류기1` | 6 | 25~28 | 25, 26, `중요순서` |
| `전기기기/2.동기기1` | 8 | 35~38 | 35~38, `4~5번 나옴`, `외우기` |
| `전기기기/3.변압기1` | 10 | 43~46 | 44~46, `자주 나옴` |
| `전력공학/11.발전1` | 9 | 38~43 | 39~41, `흡출관 중요` |
| `전력공학/4.송전특성및조상설비` | 10 | 38·40·41 등 | 38·40·41, `자주 나오는 것` |
| `회로이론/5.대칭좌표법` | 2 | 3~6 | 3~6, `시험 잘 나옴` |

표본에서는 사용자가 설명한 대로 필기 앞의 번호가 원본 문제 이미지의 번호와 일치했다. 따라서 문제번호는 글 전체보다 훨씬 강한 연결 근거로 쓸 수 있다.

## 중요 표시 탐지 규칙

### 예비 전수 검색 결과

필기 인식 후보만 대상으로 한 사전 검색 결과다. 아래 숫자는 **문제 수가 아니라 필기 페이지 엔트리 수**이며, 최종 검수 전 수치다.

| 신호 | 감지 페이지 | 감지 문서 | 주의점 |
|---|---:|---:|---|
| 긍정 빈도 문구 | 23 | 16 | `중요`, `자주 나옴`, `잘 나옴`, `필수` 등 |
| 부정·제외 문구 | 24 | 22 | `잘 안 나옴`, `삭제`, `빼자`, `하지 마` 등 |
| 암기 문구 | 68 | 30 | `외우`, `암기`, `답만` 등 |
| 별 모양 후보 글자 | 108 | 41 | 수식의 곱셈·미지수를 `☆`로 오인한 사례 포함 |
| 위 신호 중 하나 이상 | 188 | 46 | 서로 중복되는 페이지 포함 |

별 모양 후보 108개를 곧바로 108개의 중요 표시로 확정하면 안 된다. 예를 들어 필기 수식 속 기호가 `☆`로 인식된 경우도 있다. 별표는 글자 인식 결과, 실제 벡터 획 모양, 주변 문제번호, 같은 줄 여부를 함께 확인해야 한다.

### 신호 분류

#### 빈출·중요 신호

- 강한 긍정: `아주 자주 나옴`, `4~5번 나옴`, `필수`, 여러 개의 별표
- 일반 긍정: `잘 나옴`, `자주 나옴`, `많이 나옴`, `중요`, `빈출`
- 약한 긍정: `출제`, `기출`만 단독으로 적힌 경우

#### 암기 신호

- `외우자`, `암기`, `답만 외우자`, `그냥 외우는 문제`
- 암기 신호는 출제 빈도를 뜻하지 않는다. `시험에 잘 안 나오지만 답은 외우자`도 가능하므로 별도 축으로 저장한다.

#### 부정·제외 신호

- `잘 안 나옴`, `시험 안 나옴`, `출제빈도 낮음`
- `삭제`, `빼자`, `하지 마`, `과감히 빼자`
- 부정 구문이 긍정 단어보다 우선한다. 예를 들어 `출제빈도 매우 낮음`을 `출제`라는 이유로 긍정 처리하면 안 된다.

### 문제 연결 알고리즘

1. `index.notes.pb`에서 문서의 GoodNotes 페이지 순서를 복원한다.
2. 필기 `search/<UUID>`를 같은 UUID의 페이지에 연결한다.
3. 해당 필기가 PDF 직접 필기인지, PDF 뒤의 대응 필기 페이지인지 분류한다.
4. 대응 PDF 페이지 후보를 결정한다.
5. 필기 후보 3개에서 문제번호를 정규화한다.
   - `25`, `25.`, `㉕`, `②⑤`처럼 인식된 표현을 숫자 25 후보로 통일한다.
   - `O/0`, `I/1`, `S/5` 같은 OCR 혼동은 앞뒤 문맥과 PDF 문제번호 목록으로 제한한다.
6. 대응 PDF 페이지에서 실제 문제번호 목록을 추출한다.
7. 양쪽에 같은 번호가 하나만 존재하면 해당 문제로 연결한다.
8. 별표나 중요 문구의 사각형 위치를 구해 같은 줄 또는 가장 가까운 앞쪽 문제번호에 붙인다.
9. 자동 연결 근거와 원문을 `importance_signal`에 남기고, 사람이 고친 값은 별도 필드에 저장한다.

### 연결 신뢰등급

| 등급 | 조건 | 처리 |
|---|---|---|
| A | 같은 문서·대응 PDF 쪽·동일 문제번호가 모두 일치하고, 표시가 번호와 공간적으로 가깝다 | 자동 적용 가능 |
| B | 대응 PDF 쪽은 확실하지만 문제번호가 불분명하거나 한 표시가 두 문제 사이에 있다 | 검수 목록에 넣고 임시 연결 |
| C | 순수 필기 노트, 별 모양만 인식, 문제번호 불일치, 추가 페이지 오프셋 | 자동 적용하지 않고 수동 검수 |

### DB에 보존할 중요도 근거

중요도는 질문 행에 문자열 하나만 넣지 말고, 근거 행을 여러 개 보존하는 편이 안전하다.

```text
importance_signal
- id
- question_id
- source_document_id
- note_page_uuid
- note_page_index
- mapped_pdf_page_index
- handwritten_problem_number
- signal_type        // frequent, important, memorize, low_frequency, exclude, star
- signal_strength    // strong, normal, weak
- raw_text
- candidate_texts
- bbox_x, bbox_y, bbox_w, bbox_h
- mapping_confidence // A, B, C
- reviewed
- reviewer_value
```

화면에 표시하는 `중요`, `암기`, `낮은 빈도` 배지는 위 근거 행에서 계산한다. 사용자가 수정해도 원래 필기 근거는 지우지 않는다.

## 문제 중복 제거 규칙

중복 제거는 문제를 삭제하는 작업이 아니라 `question_family` 아래에 여러 출처·변형을 묶는 작업으로 설계한다.

### 완전 중복

다음이 모두 같으면 같은 문제 변형으로 묶는다.

- 정규화한 문제 본문
- 보기 내용과 순서
- 정답
- 수식의 정규형
- 그림이 있으면 이미지 지문 또는 perceptual hash

### 숫자·표현만 바뀐 유사 문제

- 숫자와 단위 값을 자리표시자로 바꾼 본문 골격이 같다.
- 필요한 공식과 풀이 절차가 같다.
- 묻는 대상과 정답 개념이 같다.

이 경우 하나를 지우지 않고 같은 `question_family`의 `variant`로 둔다. 랜덤 시험에서는 한 회차 안에 같은 family가 두 번 나오지 않게 한다.

### 합치면 안 되는 경우

- 같은 공식을 쓰지만 묻는 개념이 다르다.
- 단상과 3상, 정상상태와 과도상태처럼 적용 조건이 다르다.
- 보기가 비슷해도 정답 근거가 다르다.
- 그림이나 회로 구성이 달라 풀이가 달라진다.

### 권장 중복 판정 순서

1. 동일 출처·문제번호의 재수록 여부
2. 본문·보기의 정확 지문
3. 수식·숫자를 정규화한 템플릿 지문
4. 이미지 유사도
5. 의미 유사도와 정답 개념
6. 애매한 묶음만 사람 검수

## 공식 중복 제거 규칙

### 공식은 `family`와 `variant`로 나눈다

예를 들어 다음 두 식은 같은 공식 family의 대수적 변형이다.

```text
E = 4.44 K_w f N Φ
Φ = E / (4.44 K_w f N)
```

그러나 다음은 관련된 전력 공식이어도 적용 조건이 다르므로 별도 variant로 둔다.

```text
P = V I cosθ              // 단상
P = √3 V_L I_L cosθ       // 평형 3상
```

### 정규화 단계

1. 원본 문제 이미지와 OCR 문자열을 함께 보존한다.
2. 유니코드 기호를 통일한다: `×`, `·`, `*`, 분수선, 위첨자·아래첨자, `φ/Φ`, `θ` 등.
3. LaTeX 표시식과 계산용 AST를 분리해 저장한다.
4. 곱셈 순서, 불필요한 괄호, 부호, 분수 표현을 정규화한다.
5. 식을 대수적으로 이항한 형태는 같은 family 후보로 만든다.
6. 변수의 의미, 단위 차원, 과목·단원, 적용 조건을 비교한다.
7. 조건까지 같을 때만 자동 병합한다.

변수 이름만 보고 전 과목에서 합치면 안 된다. `E`, `V`, `I`, `P`는 단원마다 의미가 달라질 수 있으므로 `variable_definition`과 `unit_dimension`을 함께 본다.

### 문제와 공식의 관계 역할

한 문제에 보이는 모든 식을 동일한 빈도로 세면 안 된다. 관계에 역할을 둔다.

```text
question_formula
- question_id
- formula_variant_id
- role // required, answer, given, explanation, distractor, handwritten_only
- extraction_confidence
- verified
```

- `required`, `answer`: 공식 출제 빈도에 포함
- `given`: 보조 통계로만 집계
- `explanation`: 암기 노트에는 포함할 수 있지만 문제 출제 빈도에는 약하게 반영
- `distractor`: 빈도에서 제외
- `handwritten_only`: 검수 후 확정

## 이론 중복 제거 규칙

이론은 문장 표면이 아니라 다음 키로 묶는다.

- 과목과 단원
- 질문이 요구하는 핵심 개념
- 주어·관계·목적어
- 적용 조건과 예외
- 정답 개념

예를 들어 `조압수조는 수격작용을 완화한다`와 `수력발전소에서 수격작용 방지를 위해 설치하는 것은 조압수조다`는 같은 이론 family로 묶을 수 있다. 반면 조압수조의 종류별 적용 조건은 같은 family 안의 별도 variant 또는 하위 개념으로 둔다.

이론도 원문을 지우지 않고 다음을 함께 보존한다.

- 대표 암기 문장
- 출처별 원문 표현
- 맞는 진술과 자주 나오는 틀린 진술
- 예외·조건
- 연결된 문제 family 목록

## 공식·이론 빈도와 비중 집계

### 반드시 분리해서 보여 줄 수치

| 지표 | 의미 |
|---|---|
| `raw_occurrences` | 중복을 포함해 모든 출처에서 발견된 횟수 |
| `unique_question_families` | 중복·숫자 변형을 합친 뒤 실제로 몇 유형에서 쓰였는지 |
| `unique_source_documents` | 서로 다른 몇 문서에 걸쳐 나왔는지 |
| `actual_exam_hits` | 연도별 실제 기출에서 나온 횟수 |
| `important_question_hits` | 중요 필기 A/B 등급 문제에 연결된 횟수 |
| `subject_share` | 해당 과목의 고유 문제 family 중 이 공식·이론을 요구한 비율 |

주요 순위는 `raw_occurrences`가 아니라 `unique_question_families`를 기본으로 한다. 같은 문제가 강의노트, 문제집, 기출 정리본에 반복 수록됐다고 해서 공식 빈도를 세 배로 부풀리면 안 된다.

### 권장 기본 점수

가중치는 설정 화면에서 바꿀 수 있게 하고, 초기값은 단순하게 둔다.

```text
source_weight:
  실제 기출 = 2.0
  일반 문제집 = 1.0

importance_multiplier:
  강한 빈출/필수 = 1.5
  중요/자주 나옴 = 1.25
  표시 없음 = 1.0
  잘 안 나옴 = 0.5
  삭제/제외 = 0.0

weighted_hits = Σ(source_weight × importance_multiplier)
```

합산은 고유 question family 단위로 한다. 같은 family에 출처가 여러 개 있으면 `actual_exam_hits`와 출처 목록은 모두 보존하되, 일반 문제집 복제본이 점수를 반복해서 올리지 않도록 한다.

`subject_share`는 다음처럼 계산한다.

```text
subject_share =
  이 공식이 required/answer 역할인 고유 question family 수
  ------------------------------------------------------
  해당 과목의 전체 고유 question family 수
```

한 문제에 여러 공식이 필요할 수 있으므로 개별 공식의 비중 합계는 100%가 아닐 수 있다. 상위 공식 묶음이 몇 문제를 덮는지 보여 줄 때는 문제 family의 합집합으로 `coverage`를 계산한다.

## 암기 노트 생성 규칙

공식 카드의 권장 앞·뒷면은 다음과 같다.

```text
앞면
- 어떤 상황에서 쓰는 공식인가?
- 구하려는 값
- 필요한 조건

뒷면
- 대표 공식
- 변수 뜻과 단위
- 적용 조건·예외
- 같은 family의 변형식
- 실제 기출 횟수 / 고유 문제 유형 수 / 과목 내 비중
- 연결 문제 바로 풀기
```

이론 카드는 `질문 → 핵심 한 문장 → 틀리기 쉬운 반대 표현 → 관련 문제` 순서가 적합하다.

공식·이론 카드는 빈도순만 제공하지 말고 다음 필터를 함께 둔다.

- 실제 기출순
- 중요 필기순
- 고유 문제 유형 수 순
- 아직 틀린 횟수가 많은 순
- 과목·단원별

## 구현 순서

1. 원본 문서·PDF 페이지·GoodNotes 페이지 UUID를 먼저 적재한다.
2. PDF 페이지에서 문제번호별 이미지 영역을 분리하고 `questions`를 만든다.
3. 문제 본문·보기·정답을 추출하되 원본 이미지를 정답 기준 원본으로 유지한다.
4. 필기 페이지를 구조적으로 대응 PDF 쪽에 연결한다.
5. 필기 문제번호와 PDF 문제번호를 교차검증한다.
6. 중요 신호를 A/B/C 등급으로 연결한다.
7. 문제 family와 variant를 만들어 중복을 정리한다.
8. 공식·이론을 추출하고 question relation을 검수한다.
9. 고유 문제 family 기준 빈도·비중을 계산한다.
10. 랜덤 시험과 암기 노트 화면에서 같은 DB를 사용한다.

## 완료 판정 기준

- 임의 표본 100개에서 A등급 중요 표시의 문제번호 연결 정확도 98% 이상
- `잘 안 나옴`이 긍정 중요 문제로 분류되는 오류 0건
- 같은 question family가 한 모의시험에 중복 출제되는 오류 0건
- 공식 병합 표본에서 적용 조건이 다른 식의 잘못된 자동 병합 0건
- 각 공식 빈도에서 중복 수록 문제를 제외한 근거 문제 목록을 역추적 가능
- 모든 자동 교정 결과에서 원본 이미지·원문 후보·수정 이력을 확인 가능
